# SPEC: Book Editor - Comprehensive Specification

**Version:** 1.0.0
**Created:** 2026-04-14
**Project:** Book Editor untuk Academic Thesis Writing
**Framework:** SvelteKit + Svelte 5 + TypeScript + TailwindCSS v4

---

## 1. Overview

### 1.1 Purpose

Book Editor adalah aplikasi web untuk menulis buku/thesis akademik dengan dukungan:

- Markdown untuk struktur dokumen
- LaTeX untuk persamaan matematika
- Image insertion untuk figur
- Version history untuk tracking perubahan
- Export ke berbagai format

### 1.2 Target User

- Penulis thesis akademik
- Penulis buku teknis
- Researcher yang menulis dokumen kompleks

### 1.3 Key Features

1. **Book Management** - Create, edit, delete books
2. **Chapter Organization** - Hierarchical chapter/section structure
3. **Rich Content Editing** - Markdown + LaTeX + Images + Tables + Code blocks
4. **Live Preview** - Real-time rendered preview
5. **Version Control** - Auto-save, version history, restore
6. **Typography Settings** - Font family, font size
7. **Export** - Markdown, PDF

## 3. Data Model

### 3.1 Core Entities

```typescript
interface Book {
	id: string;
	title: string;
	author?: string;
	description?: string;
	createdAt: Date;
	updatedAt: Date;
	chapters: Chapter[];
	settings: BookSettings;
}

interface Chapter {
	id: string;
	bookId: string;
	title: string;
	order: number;
	content: string;
	subsections: Subsection[];
	createdAt: Date;
	updatedAt: Date;
}

interface Subsection {
	id: string;
	chapterId: string;
	title: string;
	order: number;
	content: string;
}

interface BookSettings {
	fontFamily: 'sans' | 'serif' | 'mono';
	fontSize: number;
	lineHeight: number;
	theme: 'light' | 'dark' | 'system';
	autoSaveInterval: number; // seconds
}

interface Version {
	id: string;
	bookId: string;
	timestamp: Date;
	label?: string;
	snapshot: string; // JSON serialized Book
	wordCount: number;
}

interface Asset {
	id: string;
	bookId: string;
	chapterId?: string;
	filename: string;
	mimeType: string;
	data: Blob | string;
	size: number;
	createdAt: Date;
}
```

### 3.2 IndexedDB Schema

**Database:** `book-editor-db`

**Object Stores:**

1. `books` - Key: id, Indexes: updatedAt
2. `chapters` - Key: id, Indexes: bookId, order
3. `versions` - Key: id, Indexes: bookId, timestamp
4. `assets` - Key: id, Indexes: bookId

## 4. Functionality Specification

### 4.1 Book Management

**Create Book:**

1. User clicks "New Book" button
2. Modal appears with title input
3. User enters title, clicks Create
4. Book created with empty chapter list
5. User redirected to editor view

**Edit Book:**

1. User clicks book title in header
2. Title becomes editable input
3. User types, blurs or presses Enter to save
4. Book updated in IndexedDB

**Delete Book:**

1. User right-clicks book in list OR uses menu
2. Confirmation dialog appears
3. User confirms deletion
4. Book and all chapters/versions deleted

**Open Book:**

1. User clicks book in home list
2. Book data loaded from IndexedDB
3. Editor view opens with first chapter

### 4.2 Chapter Management

**Create Chapter:**

1. User clicks "+" button in outline
2. New chapter created with default title "Chapter N"
3. Chapter added to end of list
4. Editor focuses on new chapter

**Edit Chapter Title:**

1. User double-clicks chapter in outline
2. Title becomes editable input
3. User types, blurs to save
4. Chapter updated in state and IndexedDB

**Reorder Chapters:**

1. User drags chapter via handle
2. Visual indicator shows drop position
3. On drop, chapter order updated
4. All affected chapters re-indexed

**Delete Chapter:**

1. User right-clicks chapter → Delete
2. Confirmation dialog with warning
3. User confirms
4. Chapter removed from book

### 4.3 Content Editing

**Markdown Editing:**

- CodeMirror 6 with Markdown language support
- Syntax highlighting for:
  - Headers (# ## ###)
  - Bold/Italic (\* \*\*)
  - Links [text](url)
  - Images ![alt](url)
  - Code blocks (```)
  - Math delimiters ($ $$)

**Toolbar Actions:**
| Button | Action | Shortcut |
|--------|--------|----------|
| B | Wrap selection in \*_ | Ctrl+B |
| I | Wrap selection in _ | Ctrl+I |
| H | Insert heading structure | |
| img | Open image picker | |
| table | Insert table template | |
| math | Insert math block $$ | Ctrl+Shift+M |
| code | Insert code block ``` | |

**Auto-save:**

1. User types in editor
2. Debounce timer starts (30 seconds)
3. If no more typing, auto-save triggers
4. Book state serialized and saved
5. Version created if significant change

**Images:**

- Rendered with max-width: 100%

**Tables:**

- Bootstrap table styles

**Code Blocks:**

- Syntax highlighted with highlight.js

**Cross-references:**

- `Gambar 4.1` → clickable link to figure
- `Tabel 4.2` → clickable link to table

### 4.5 Version History

**Version List:**

**Restore Version:**

**CodeMirror Setup:**

```typescript
import { EditorView, keymap, lineNumbers, highlightActiveLine } from '@codemirror/view';
import { EditorState } from '@codemirror/state';
import { markdown, markdownLanguage } from '@codemirror/lang-markdown';
import { syntaxHighlighting, defaultHighlightStyle } from '@codemirror/language';

const extensions = [

	markdown({ base: markdownLanguage }),
	syntaxHighlighting(defaultHighlightStyle),

	EditorView.lineWrapping
];
```

**KaTeX Rendering:**

```typescript
import katex from 'katex';

export function renderMath(text: string): string {
	// Match inline math $...$ and block $$...$$
	return text
		.replace(/\$\$(.*?)\$\$/g, (_, math) =>
			katex.renderToString(math, { displayMode: true, throwOnError: false })
		)
		.replace(/\$(.*?)\$/g, (_, math) =>
			katex.renderToString(math, { displayMode: false, throwOnError: false })
		);
}
```

**IndexedDB Operations:**

```typescript
import { openDB, type DBSchema, type IDBPDatabase } from 'idb';

interface BookEditorDB extends DBSchema {
	books: { key: string; value: Book; indexes: { 'by-updated': Date } };
	chapters: { key: string; value: Chapter; indexes: { 'by-book': string } };
	versions: { key: string; value: Version; indexes: { 'by-book': string } };
	assets: { key: string; value: Asset; indexes: { 'by-book': string } };
}

export async function initDB() {
	return openDB<BookEditorDB>('book-editor-db', 1, {
		upgrade(db) {
			const bookStore = db.createObjectStore('books', { keyPath: 'id' });
			bookStore.createIndex('by-updated', 'updatedAt');

			const chapterStore = db.createObjectStore('chapters', { keyPath: 'id' });
			chapterStore.createIndex('by-book', 'bookId');

			const versionStore = db.createObjectStore('versions', { keyPath: 'id' });
			versionStore.createIndex('by-book', 'bookId');

			const assetStore = db.createObjectStore('assets', { keyPath: 'id' });
			assetStore.createIndex('by-book', 'bookId');
		}
	});
}
```

**Auto-save Effect:**

```typescript
$effect(() => {
	if (!currentChapter) return;

	const timer = setTimeout(() => {
		saveChapter(currentChapter);
		createVersion(currentBook.id);
	}, 30000); // 30 second debounce

	return () => clearTimeout(timer);
});
```

## 7. Error Handling

**IndexedDB Errors:**

- Storage full: Show warning, suggest export
- Transaction failed: Retry with exponential backoff
- Data corrupted: Offer restore from version

**Rendering Errors:**

- Invalid LaTeX: Show error placeholder, keep original text
- Broken image: Show placeholder with filename
- Invalid markdown: Render as plain text

**Network Errors:**

- Export failed: Show error toast, retry button
- N/A for local-only app
