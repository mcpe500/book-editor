# SPEC: AI Chat Feature for Book Editor

**Version:** 1.0.0
**Created:** 2026-04-15
**Project:** Book Editor - AI Writing Assistant
**Framework:** SvelteKit + Svelte 5 + TypeScript + TailwindCSS v4

---

## 1. Overview

### 1.1 Purpose

Add AI chat feature to book editor enabling users to:

- Get AI assistance for writing book content
- Receive multiple response options (A, B, C) to choose from
- Configure AI behavior via cursor rules
- Track version history of AI interactions

### 1.2 Target User

- Authors writing thesis/books with AI assistance
- Users who want choice between multiple AI suggestions
- Users who need consistent AI behavior via custom rules

## 5. Functionality Specification

### 5.1 Core Features

#### F1: Chat Interaction

1. User types message in input field
2. User clicks Send or presses Enter
3. Message appears in chat (user style)
4. AI "typing" indicator shows
5. Streaming response appears in real-time
6. When complete, options appear as cards

#### F2: Selectable Response Options

1. AI generates 2-3 response alternatives
2. Each option shown as labeled card (A, B, C)
3. User can:
   - Hover to preview full text
   - Click "Insert" to add to editor at cursor
   - Dismiss all without inserting
4. Only one option can be selected per request

#### F3: Cursor Rules

1. User adds custom rules that guide AI behavior
2. Rules are included in system prompt
3. Rules can be toggled on/off individually
4. Rules persist across sessions
5. Examples:
   - "Write in formal Indonesian academic style"
   - "Use LaTeX for all mathematical equations"
   - "Follow kerangka-ver2.md structure"

#### F4: Version History

1. Each chapter maintains its AI conversation history
2. User can view past conversations
3. User can restore/continue a past conversation
4. User can clear history

#### F5: Editor Integration

1. "AI Assistant" button in editor toolbar
2. Click opens AI chat panel
3. Insert text at current cursor position
4. Works across all chapters

### 5.2 System Prompt Construction

```typescript
function buildSystemPrompt(cursorRules: CursorRule[], chapterContext: string): string {
	const enabledRules = cursorRules.filter((r) => r.enabled);

	let prompt = `You are an AI writing assistant for a book editor.
  
The user is writing a book/thesis. Help them with:
- Writing and editing content
- Suggesting alternatives
- Formatting (Markdown, LaTeX)
- Following structure guidelines

`;

	if (enabledRules.length > 0) {
		prompt += `\nCURSOR RULES (follow these instructions):\n`;
		enabledRules.forEach((rule, i) => {
			prompt += `${i + 1}. ${rule.content}\n`;
		});
	}

	if (chapterContext) {
		prompt += `\nCURRENT CHAPTER CONTENT:\n${chapterContext}\n`;
	}

	prompt += `\nWhen providing writing suggestions, provide 2-3 different options (labeled A, B, C) that the user can choose from.
Each option should be a complete alternative approach to the user's request.`;

	return prompt;
}
```

### 5.3 User Flows

#### Flow 1: Get AI Help with Writing

1. User selects chapter
2. User clicks AI Assistant button
3. Chat panel opens
4. User types: "Write an introduction for this chapter about quantum computing"
5. AI responds with 3 options
6. User reads previews
7. User clicks "Insert B"
8. Text inserted at cursor in editor
9. Panel stays open for more requests

#### Flow 2: Configure Cursor Rules

1. User opens AI chat
2. User clicks "Cursor Rules" expand
3. User clicks "+ Add rule"
4. User types: "Use formal Indonesian academic language"
5. Rule added and enabled
6. Next AI request uses this rule

#### Flow 3: Review Past Conversation

1. User clicks "History" button
2. History sidebar slides in
3. User sees list of past conversations
4. User clicks one to load it
5. Conversation restored in main view

## 9. Error Handling

### 9.1 API Errors

- **No API Key:** Show setup instructions
- **Rate Limited:** Show retry timer
- **Network Error:** Show offline message, disable send
- **Empty Response:** Show "No suggestions generated"

### 9.2 UI Errors

- **Insert Failed:** Show error toast, keep text in clipboard
- **Save Failed:** Show warning, allow retry
