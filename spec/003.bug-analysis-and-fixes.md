## 2. Critical Bugs Fixed

### 2.1 SettingsModal Theme Dropdown Bound to Wrong Store

**Severity:** Critical
**File:** `src/lib/components/Settings/SettingsModal.svelte`
**Line:** ~104-112

**Bug:** Theme dropdown `value={settingsStore.fontFamily}` instead of `value={settingsStore.theme}`. User could never change theme.

**Fix:** Changed binding to `value={settingsStore.theme}`.

**Testing:** Open Settings → Change Theme → Verify dropdown shows correct selection → Reload page → Theme persists.

---

### 2.2 Header Title Edit Not Persisting to IndexedDB

**Severity:** Critical
**File:** `src/lib/components/Layout/Header.svelte`
**Lines:** 1-31

**Bug:** `saveTitle()` updated `bookStore.currentBook.title` but never called `updateBook()` to persist to IndexedDB. Title changes lost on reload.

**Fix:** 
- Imported `updateBook` from `$lib/db/books`
- Made `saveTitle()` async and added `await updateBook(bookStore.currentBook)`
- Changed `startEditTitle()` to read from `bookStore.currentBook?.title` instead of prop

**Testing:** Click title → Edit → Press Enter → Reload page → Title persists.

---

### 2.3 Auto-Save Debounce Creating Multiple Timers Without Cleanup

**Severity:** Critical
**File:** `src/routes/book/[id]/+page.svelte`
**Lines:** 30-45, 139-149

**Bug:** `$effect(() => { if (bookStore.hasUnsavedChanges) { handleAutoSave() } })` created a new timer on every `hasUnsavedChanges` change without clearing the previous one. Multiple concurrent timers could fire, causing duplicate saves and version creation.

**Fix:**
- Renamed `handleAutoSave` to `scheduleAutoSave`
- Added proper cleanup in `onMount` return function: `clearTimeout(autoSaveTimer)`
- Added `beforeunload` event listener for unsaved changes warning

**Testing:** Make changes → Wait 30s → Verify only one version created → Navigate away with unsaved changes → Browser shows confirmation dialog.

---

### 2.4 Version Restore Not Updating Chapters from Snapshot

**Severity:** Critical
**File:** `src/lib/db/versions.ts`
**Lines:** 43-65

**Bug:** `restoreVersion()` only restored the book metadata, not the associated chapters. After restore, book would reference chapters that no longer exist or have wrong content.

**Fix:** 
- Changed return type to `{ book: Book; chapters: Chapter[] }`
- Now fetches and returns chapters associated with the restored book
- Caller can use this to update store state

**Testing:** Create version → Edit chapters → Restore version → Verify chapters match version snapshot.

---

### 2.5 AI sendMessage Race Condition Creating Duplicate Conversations

**Severity:** Critical
**File:** `src/lib/stores/ai.svelte.ts`
**Lines:** 140-186

**Bug:** Pattern `if (!aiStore.currentConversation) { startNewConversation() }` followed by another `if (!aiStore.currentConversation) return;` is fragile. If `startNewConversation` fails, the code continues and crashes.

**Fix:** Simplified flow. Changed `saveCurrentConversation()` to `saveConversations(aiStore.conversations)` to directly persist the updated conversation list.

**Testing:** Open AI panel → Send message → Verify single conversation created → Check localStorage → Only one conversation stored.

---

### 2.6 handleInsertOption Not Persisting Selected State

**Severity:** Moderate → Critical
**File:** `src/lib/stores/ai.svelte.ts`
**Lines:** 304-312

**Bug:** `insertOption()` marked option as `selected: true` for ALL options instead of the specific one selected. Also didn't persist to localStorage.

**Fix:**
- Changed to `selected: o.content === content` (only the chosen option)
- Added `saveConversations(aiStore.conversations)` to persist

**Testing:** Insert option B → Reload page → Load conversation → Option B shows "Selected".

---

### 2.7 ChatMessage Not Rendering Markdown in AI Messages

**Severity:** Critical
**File:** `src/lib/components/AI/ChatMessage.svelte`
**Lines:** 1-36

**Bug:** AI messages displayed as plain text `{message.content}`. No markdown, LaTeX, or formatting rendered.

**Fix:**
- Imported `parseMarkdown` from `$lib/utils/markdown`
- Added `$derived` to render markdown for assistant messages
- Changed to `{@html renderedContent}` for AI messages

**Testing:** Ask AI to write content with headings/math → Verify formatted output.

---

### 2.8 Markdown Processing Pipeline Order

**Severity:** Moderate
**File:** `src/lib/utils/markdown.ts`
**Lines:** 119-132

**Bug:** `processCodeBlocks` and `processTables` ran BEFORE `marked.parse()`. This caused double-processing: custom pre-processing wrapped content in HTML, then `marked.parse()` tried to parse it again, creating broken output.

**Fix:** Removed custom `processTables` and `processCodeBlocks` from pre-processing pipeline. `marked` handles tables and code blocks natively with better accuracy.

**Testing:** Write markdown with tables and code blocks → Verify correct HTML output.

---

### 2.9 Code Blocks Missing Syntax Highlighting

**Severity:** Moderate
**File:** `src/lib/utils/markdown.ts`
**Lines:** 1-14, 131-159

**Bug:** `highlight.js` was imported but never used. Code blocks rendered as plain text without color.

**Fix:**
- Imported `hljs` core and registered languages (typescript, python, javascript)
- Created custom `marked.Renderer` with `code` callback
- Used `hljs.highlight()` for specified language, `hljs.highlightAuto()` as fallback

**Testing:** Write ````typescript ... ```` → Verify syntax highlighted output.

---

### 2.10 Dark Mode CSS Missing

**Severity:** Feature
**Files:** `src/routes/layout.css`, `src/lib/stores/settings.svelte.ts`

**Bug:** Theme setting existed (light/dark/system) but no CSS to support it.

**Fix:**
- Added CSS custom properties (`--bg-primary`, `--text-primary`, etc.) for light and dark themes
- Added `[data-theme='dark']` and `[data-theme='light']` selectors
- Added `@media (prefers-color-scheme: dark)` for system theme
- Added `applyTheme()` function to settings store that sets `data-theme` attribute

**Testing:** Settings → Change to Dark → Verify background/text colors change → Change to System → Verify follows OS preference.

---

### 2.11 Beforeunload Confirmation Missing

**Severity:** Moderate
**File:** `src/routes/book/[id]/+page.svelte`
**Lines:** 30-45

**Bug:** User could navigate away with unsaved changes without warning. Data loss risk.

**Fix:** Added `beforeunload` event listener in `onMount` that checks `bookStore.hasUnsavedChanges` and shows browser confirmation dialog.

**Testing:** Make changes → Close tab → Browser shows "Unsaved changes" confirmation.

---

### 2.12 Markdown Renderer Type Signature Fix

**Severity:** Build Error
**File:** `src/lib/utils/markdown.ts`
**Lines:** 136

**Bug:** `renderer.code = (code, language) => {` had wrong signature for marked v18. Expected `({ text, lang })` object parameter.

**Fix:** Changed to `renderer.code = ({ text, lang }: { text: string; lang?: string }) => {`

**Testing:** `bun run check` passes with 0 errors.

## 6. Architecture Observations

### 6.1 Strengths

- **Svelte 5 runes** used correctly throughout (`$state`, `$derived`, `$effect`)
- **Clean separation** of concerns (db, stores, services, components)
- **TypeScript** strict mode enabled
- **IndexedDB** via `idb` provides offline-first architecture
- **Streaming AI responses** properly implemented with AbortController

### 6.2 Weaknesses

- **No error boundaries** for async operations
- **No loading states** for DB queries
- **No retry logic** for failed operations
- **No input validation** on user-entered data
- **No rate limiting** on AI requests
- **localStorage for AI data** may hit 5MB limit with many conversations
